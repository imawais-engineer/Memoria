Primary data flows

  1. Frontend → Backend: User message via POST /chat or POST /chat/stream from the React dashboard.
  2. Backend → Database: Agent reads Session Memory from Redis; retrieves Personal Memories from PostgreSQL + pgvector (scoped by PI / MemoryLess).
  3. Backend → Qwen Cloud: Packed memory context + persona + product knowledge + message sent to Qwen; query embedded with text-embedding-v3.
  4. Qwen → Backend → Frontend: Personalized reply (Markdown/LaTeX) streams or returns through FastAPI.
  5. Async pipeline: Celery worker extracts memories via Qwen function calling → embed → store; Beat jobs decay and consolidate on schedule.
  6. MCP: External agents discover tools at GET /mcp/memory-skills and invoke memory operations with ownership checks.
