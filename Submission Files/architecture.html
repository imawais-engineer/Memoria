Primary data flow — how Qwen Cloud connects to backend, database, and frontend

  1. Frontend → Backend: User message via POST /chat from the React dashboard.
  2. Backend → Database: Agent reads Session Memory from Redis and retrieves Personal Memories from PostgreSQL + pgvector.
  3. Backend → Qwen Cloud: Agent sends packed context + message to Qwen-Plus; embeds the query with text-embedding-v3.
  4. Qwen Cloud → Backend → Frontend: Personalized reply returns through FastAPI and renders in chat (Markdown / LaTeX).
  5. Backend → Celery → Qwen → Database: Async workers call Qwen-Plus / Qwen-Max to extract, consolidate, and write memories back to PostgreSQL.
